Deep Compressor

将超长金融文本 (8K–64K tokens) 压缩为固定长度的隐向量序列，作为冻结 Qwen 解码器的前缀， 回答细节级金融问答。通过 Teacher Distillation 保证压缩前缀携带等效于完整文档的信息。

Qwen3-0.6B ~ 8B Perceiver Compression Teacher Distillation Sequence-Length-Only

Core Design: Sequence-Length-Only Compression

对于已知 Qwen3 模型，perceiver_dim 被强制等于 qwen.hidden_size (如 Qwen3-0.6B 为 1024)。DownProj / UpProj 默认为 Identity（无参数）—— 压缩仅通过序列长度缩减实现 (doc_len → num_queries)，而非维度缩减。 MLP/Linear 模式仅用于消融实验。

Module Details

🧊

Qwen Encoder

Frozen

调用 self.qwen.model(...) (内部 Transformer)，不是 self.qwen(...)。 后者会计算 LM Head logits (~15GB)，导致 OOM。

运行在 torch.no_grad() 中。输出 last_hidden_state 后立即 .detach()。

(B, seq_len) → (B, seq_len, hidden_size)
⬇

DownProj

Default: Identity

将 Qwen hidden states 投影到 Perceiver 维度。由于 perceiver_dim == qwen.hidden_size，默认为 Identity（无参数）。

三种模式用于消融实验：

ModeStructure
identity直通，无参数
linearLinear + LayerNorm
mlpLinear → GELU → Linear → LN
🔍

QueryInit

Trainable

可学习基础查询 + 问题条件偏置。

base_queries: nn.Parameter(randn(Q, D) * 0.02)

question_proj: Linear(qwen_dim, perceiver_dim)

NTP 阶段 (Stage 1): 问题向量为零 → 使用纯基础查询。
QA 阶段 (Stage 2): 问题经 mean pool → 线性投影为 additive bias。

(B, qwen_dim) → (B, Q, perceiver_dim)
🗜

GuidedPerceiver

Trainable

三阶段注意力压缩架构，将长文档序列压缩为固定长度隐向量。每个 Stage 可通过 AblationConfig 独立开关。

Stage A — Global Compression
Cross-Attention (2 layers)
queries ← byte_array
Self-Attention (2 layers)
queries 内部交互

从文档中提取全局信息
Stage B — Self Refinement
Self-Attention (2 layers)
queries 之间的信息融合


整合与精炼已提取的信息
Stage C — Deep Reasoning
Cross-Attention (2 layers)
回读 byte_array 获取细节
Self-Attention (4 layers)
深层推理处理
补充细节，深层推理
queries (B, Q, D) + byte_array (B, doc_len, D) → latent_array (B, Q, D)
⬆

UpMLP

Default: Identity

将 Perceiver 维度映射回 Qwen 维度。默认 Identity，结构与 DownProj 对称。

输出作为 prefix embeddings，与 question+answer token embeddings 拼接后送入 Qwen Decoder。

(B, Q, perceiver_dim) → (B, Q, qwen_dim)
🧊

Qwen Decoder

Frozen

调用 self.qwen(inputs_embeds=...) 包含 LM Head，计算 next-token logits。

输入: [prefix_embeds | q_embeds | a_embeds]
Labels: [-100 (prefix) | -100 (question) | answer_labels]

注意: Qwen forward(labels=...) 内部自动 shift，不要手动 shift。

Training Pipeline

① Stage 1: NTP Pretraining

  • 任务：压缩文档 → 前缀 → Next Token Prediction
  • 问题向量为零（无问题条件）
  • 损失：标准 Cross-Entropy
  • 目标：学习通用文档压缩能力
  • 可选：Question-Guided Compression，训练时加入问题上下文

② Stage 2: QA Fine-tuning + Distillation

  • 任务：压缩文档 → 前缀 + 问题 → 回答
  • 问题条件查询初始化激活
  • Teacher 看完整未压缩文档 + 问题 + 答案
  • Student 看压缩前缀 + 问题 + 答案
  • 三项损失：QA CE + KL Distill + Hidden MSE

Loss Functions

Ltotal = λce · LQA-CE + λkl · LKL + λmse · LHidden-MSE

QA Cross-Entropy

标准自回归语言模型损失。仅对 answer 位置计算，prefix 和 question 位置标记为 -100。

Stage 1 (NTP): 对续写文本位置计算。
Stage 2 (QA): 对答案 token 位置计算。

Default weight: λce = 1.0

KL Distillation

Student 与 Teacher 输出分布的 KL 散度。对 answer mask 位置计算，使用温度缩放。

T = 2.0，最终乘以 T² (distillation convention)。

Default weight: λkl = 0.2

Hidden State MSE

选定解码器层 ([7, 14, 21, 27]) 的 student/teacher hidden state 之间的 MSE 损失。

对 question+answer 共享位置计算。权重线性 ramp：前 2000 步从 0 逐步上升到 1，避免早期训练不稳定。

Default weight: λmse = 0.0 (默认关闭)

Gradient Flow

Frozen vs Trainable

所有 Qwen 参数 requires_grad=False。Frozen Qwen 前向传播在 torch.no_grad() 上下文中运行。

可训练模块：QueryInit、GuidedPerceiver；消融时可选 DownProj、UpMLP。

这些可训练模块在 no_grad 上下文外部调用，以保留梯度流。Checkpoint 仅保存可训练权重（跳过 qwen.* 参数）。

Gradient checkpointing 需要 use_reentrant=False，否则梯度静默中断。
