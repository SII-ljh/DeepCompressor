# 4096 Tokens配置说明

## 📊 配置变更

### 核心改动

**文档长度**: 512 → **4096** (8倍)

这个改动：
- ✅ 覆盖95%+的数据样本（避免截断）
- ✅ 符合项目目标（超长文档压缩）
- ✅ 保留完整文档信息，不丢失答案

---

## ⚙️ 优化后的训练参数

### 配置对比表

| Q值 | max_doc | Batch/卡 | 累积 | 有效Batch | 预估显存 | Ckpt |
|-----|---------|----------|------|-----------|----------|------|
| **64**  | 4096 | 8  | 2 | 128 | ~60GB  (42%) | OFF |
| **128** | 4096 | 6  | 2 | 96  | ~90GB  (63%) | OFF |
| **256** | 4096 | 4  | 2 | 64  | ~120GB (84%) | OFF |
| **512** | 4096 | 2  | 2 | 32  | ~140GB (98%) | **ON** |

**说明**:
- **有效Batch**: 8卡 × 每卡Batch × 梯度累积
- **预估显存**: 基于143GB总显存的占用估算
- **Ckpt**: Gradient Checkpointing（节省显存，Q=512必须开启）

---

## 🎯 为什么这样配置？

### 原理

**文档长度增加8倍 (512→4096)**:
- 激活值显存: 增加约6-8倍
- 需要减小batch size以适应显存

### 保守策略

配置偏保守，留有余量：
- Q=64/128/256: 有20-40%显存余量
- Q=512: 接近上限，启用gradient checkpointing

### 如果显存充足

训练时监控`nvidia-smi`，如果显存占用<70%，可以：

```bash
# 增加batch size（例如Q=256）
# 编辑 configs/qa_q256_8gpu.yaml:
training:
  batch_size: 6   # 4 → 6
```

## ⚠️ 注意事项

### Q=512 必须启用Gradient Checkpointing

配置中已自动启用：
```yaml
training:
  gradient_checkpointing: true  # Q=512必须开启
```

**作用**: 节省30-40%显存，但训练慢15-20%

### 如果还是OOM

**方案1**: 减小batch size
```yaml
# configs/qa_q256_8gpu.yaml
training:
  batch_size: 2    # 4 → 2
  gradient_accumulation_steps: 4  # 保持有效batch=64
```

**方案2**: 启用所有模型的gradient checkpointing
```yaml
training:
  gradient_checkpointing: true
```

**方案3**: 减小文档长度
```yaml
qwen:
  max_doc_tokens: 2048  # 4096 → 2048
```
